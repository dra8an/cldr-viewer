## Documentation Structure

### Naming Conventions

**PHASE-*.md** - Phase completion reports aligned with CLDR-VIEWER-IMPLEMENTATION-PLAN.md
- Format: `PHASE-{number}-{subphase?}-{title}.md`
- Examples:
  - `PHASE-1-COMPLETE.md`
  - `PHASE-2-COMPLETE-SUMMARY.md`
  - `PHASE-2-3-COMPLETE.md`
  - `PHASE-3-1-EDIT-MODE-TOGGLE.md`

**FEATURE-*.md** - Feature documentation
- Format: `FEATURE-{feature-name}.md`
- Examples:
  - `FEATURE-METADATA-PANEL-NAVIGATION.md`
  - `FEATURE-SECTION-ICONS.md`

**BUGFIX-*.md** - Bug fix reports
- Format: `BUGFIX-{issue-description}.md`
- Examples:
  - `BUGFIX-SCRIPT-TERRITORY-DETECTION.md`

**SESSION-SUMMARY-*.md** - Session summaries
- Format: `SESSION-SUMMARY-{date}-{optional-suffix}.md`
- Examples:
  - `SESSION-SUMMARY-2024-12-31-FINAL.md`

**DAILY-STATUS-*.md** - Daily status reports
- Format: `DAILY-STATUS-{date}.md`
- Examples:
  - `DAILY-STATUS-2025-12-30.md`
